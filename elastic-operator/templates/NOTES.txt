1. The Elastic Stack has been deployed with the following components:

{{- if .Values.operator.enabled }}
   - ECK Operator
{{- end }}
{{- if .Values.elasticsearch.enabled }}
   - Elasticsearch cluster with dedicated node roles
{{- end }}
{{- if .Values.kibana.enabled }}
   - Kibana
{{- end }}
{{- if .Values.filebeat.enabled }}
   - Filebeat
{{- end }}

2. Get the Elasticsearch credentials:
   kubectl get secret {{ .Values.elasticsearch.name }}-es-elastic-user -n {{ .Release.Namespace }} -o go-template='{{`{{.data.elastic | base64decode}}`}}'

3. Access your cluster:
   - Elasticsearch: https://{{ .Values.elasticsearch.name }}-es-http.{{ .Release.Namespace }}.svc:9200
   - Kibana: https://{{ .Values.kibana.name }}-kb-http.{{ .Release.Namespace }}.svc:5601

Note: The services are exposed via ClusterIP by default. To access them externally, you may need to:
1. Configure an Ingress
2. Use port-forwarding:
   kubectl port-forward service/{{ .Values.elasticsearch.name }}-es-http 9200 -n {{ .Release.Namespace }}
   kubectl port-forward service/{{ .Values.kibana.name }}-kb-http 5601 -n {{ .Release.Namespace }}
